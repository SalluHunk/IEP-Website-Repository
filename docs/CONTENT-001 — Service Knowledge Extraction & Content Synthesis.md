# OPERATION PHOENIX

## CONTENT-001 — Service Knowledge Extraction & Content Synthesis

Mission Type

Knowledge Synthesis

This mission extracts and synthesizes service content from verified project sources into the new approved nine-service catalogue.

This is NOT copywriting.

This is NOT marketing.

This is evidence-based content synthesis.

--------------------------------------------------

Mission Context

The client has approved the following nine-service catalogue.

The CMS architecture has already been updated to support it.

The deployment package is waiting only for service content.

The objective is to minimise future client effort by extracting every possible piece of verified information from existing project documents before requesting additional input.

--------------------------------------------------

Authoritative Sources

Use ONLY verified project sources, including:

• Signed Statement of Work
• Commercial Proposal
• IEP Brochure
• Client Questionnaire
• Homepage Narrative
• Client Review Notes (CRN-001)
• Any approved repository documents that contain verified capability descriptions

Do NOT use assumptions.

Do NOT invent services.

--------------------------------------------------

3.

5.

Industries

Gap

CLIENT-QUESTIONS

After all extraction is complete,

prepare a concise client review document.

Only include questions that genuinely cannot be answered from existing evidence.

Group them by service.

Aim to minimise client workload.

--------------------------------------------------

Repository Rules

Do not write marketing copy.

Do not exaggerate capability.

Do not duplicate identical text across multiple services unless genuinely justified.

Prefer evidence over completeness.

Flag uncertainty rather than filling gaps.

--------------------------------------------------

Mission Success

The mission is successful when:

The project team can review all nine services internally.

The client receives only targeted clarification requests.

The CMS deployment package can be completed with minimal additional client input.

Materialize all documents.

Stop.
