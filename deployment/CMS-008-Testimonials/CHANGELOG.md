- Initial deployment package. Migrates the 4 real, already-live testimonials (Ultra Tough Ltd, York Handmade, Harsco Environmental, Naylor Industries Plc) into a CMS-editable `cspt-testimonial` CPT.

- Full migration table for all 4 real testimonials, sourced verbatim from the live, already-approved Testimonials page (1062).
- Noted, not wired in: two of the four clients (Ultra Tough, York Handmade) have real signed recommendation letter PDFs already in the Media Library — corroborating evidence found during content sourcing, flagged in `DECISIONS.md` as a possible future feature, out of this module's scope.

## v1.2 (2026-07-19) — wp-admin editing experience audit + fixes
User reported the wp-admin edit screen looked broken: the main title/content box showed plugin demo filler text ("Cut down on all your work..."), the featured image showed a dummy stock photo, and no field for Person Name/Role was visible anywhere. Investigated properly rather than guessing:
1. **Real issue, fixed:** `cspt-testimonial` belongs to the Ultimate Addons for Elementor plugin, which has its own native edit-screen fields (title, block-editor `post_content`, featured image) entirely separate from this package's ACF fields. `[iep_testimonial_grid]` only ever read the ACF fields (so the front end was already correct), but the plugin's own native fields were never synced, leaving the edit screen showing stale demo content. Added `content` and `featured_media` handling to the authenticated endpoint and synced both to match the real data for all 4 testimonials — `featured_media` set to the person's real photo where one exists (Harsco Environmental only), cleared (not backfilled with a mismatched logo) for the other 3.
2. **False alarm, not a bug:** the "Testimonial Details" ACF box appeared to be completely missing from the edit screen. Diagnosed via a temporary read-only endpoint calling `acf_get_field_groups()`/`acf_get_fields()` directly — ACF confirmed the field group was correctly registered, active, and matched to every post, with all 6 real fields present. Root cause turned out to be a collapsed "Meta boxes" panel divider sitting at the very bottom of the Gutenberg screen, below the fold — not a registration, location-rule, or code problem at all.
3. **Real, useful discovery from the same diagnostic:** the theme already has its own native ACF field group on this CPT ("Greenly - Testimonial Details" — Company Name, Testimonial Title, Link on Company Name, Star Ratings). Confirmed it does **not** overlap with this package's fields (no quote text, no person name/role/photo) — so `group_iep_testimonial_v1` is additive, not a duplicate, consistent with the repository's standing rule against parallel/duplicate structures. The native group's fields still carry stray demo data (e.g. "Company Name: CEO of Bata") on the migrated posts — harmless since nothing reads them, flagged to Mission Control for optional cleanup rather than touched unprompted.
